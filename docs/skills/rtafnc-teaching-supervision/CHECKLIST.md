# QA Checklist

## Smoke test

- [ ] เปิดระบบได้โดยไม่มี blank page
- [ ] ตรา วพอ. แสดงใน Sidebar, Dashboard และรายงาน
- [ ] ชื่อระบบเต็มถูกต้องทุกจุดสำคัญ
- [ ] ทุกเมนูเปิดได้
- [ ] ไม่มี console error ระดับ blocking

## Form and validation

- [ ] เลือกผู้รับการนิเทศแล้วข้อมูลเติมถูกต้อง
- [ ] ห้ามเลือกผู้รับเป็นผู้ประเมินคนเดียวกัน
- [ ] ระดับประเมินครบ 3 ตัวเลือก
- [ ] คะแนนต่ำกว่า 1 หรือสูงกว่า 5 ถูกปฏิเสธ
- [ ] ส่งไม่ได้เมื่อคะแนนไม่ครบ 12 ข้อ
- [ ] บันทึกร่างได้เมื่อยังไม่ครบ
- [ ] ป้องกันรายการซ้ำในรอบ/คน/ระดับเดียวกัน

## Calculation

- [ ] ค่าเฉลี่ยด้านที่ 1/2 และรวมถูกต้อง
- [ ] ใช้เฉพาะ submitted
- [ ] ปัดทศนิยม 2 ตำแหน่งด้วย ROUND
- [ ] ไม่มี division by zero
- [ ] Dashboard ตรงกับรายการต้นทาง

## Attachments and permissions

- [ ] รับ JPG/PNG/WebP ไม่เกิน 5 MB และ 2 รูป
- [ ] Preview/Remove ทำงาน
- [ ] ผู้ประเมินแก้เฉพาะของตน
- [ ] ผู้ตรวจสอบแก้ไม่ได้
- [ ] Admin action มี Audit Log

## Completion, print and mobile

- [ ] สถานะ 0/3 ถึง 3/3 ถูกต้อง; draft ไม่นับ
- [ ] Filter ทำงาน
- [ ] A4 ไม่เกิดหน้าว่าง/ตัดแถว/แยกลายเซ็น
- [ ] ตรา ชื่อหน่วย คะแนน และรูปถูกต้อง
- [ ] Mobile 360px ใช้งาน flow หลักได้

## Release gate

ห้ามประกาศ “พร้อมใช้งานจริง” หากยังเป็น localStorage, ไม่มี auth จริง หรือยังไม่ได้ทดสอบ server-side; ให้ใช้คำว่า “MVP/Prototype” เท่านั้น
