# Repo Release Checklist

Use before adding any material to a public repository.

## Safe Material
- Generic workflow notes
- Templates
- Checklists
- Prompt libraries
- Example files with fake sample data
- Architecture notes without private details

## Hold for Internal Storage
- Real learner records
- Real assessment results
- Real staff records
- Internal official files
- Raw exports from Drive
- Credentials or configuration secrets

## Review Items
- [ ] Uses generic examples only
- [ ] No private records
- [ ] No credentials
- [ ] No raw internal evidence
- [ ] Markdown is reusable
- [ ] Use a pull request when unsure
