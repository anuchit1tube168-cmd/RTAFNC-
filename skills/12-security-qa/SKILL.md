# Skill: Security and Quality Assurance

## หน้าที่
ทดสอบระบบก่อนเปิดใช้งานจริงและหลังทุกการเปลี่ยนแปลงสำคัญ

## Security Tests
- นักเรียน A อ่านหรือแก้ข้อมูลนักเรียน B ไม่ได้
- ผู้ตรวจความดีเข้าถึงข้อมูลโรงพยาบาลไม่ได้
- เจ้าหน้าที่แพทย์เห็นเฉพาะข้อมูลตาม Scope
- ผู้ไม่มีสมาชิกเปิดลิงก์แล้วไม่เห็นข้อมูลส่วนตัว
- ไฟล์ Private เปิดไม่ได้เมื่อไม่มี Session
- URL ไฟล์หมดอายุตามกำหนด
- ค่าฝั่ง Server ไม่ปรากฏใน Frontend หรือ Log

## Functional Tests
- เปิด LIFF จาก Rich Menu
- Login และผูกสมาชิก
- ส่งความดีทั้ง 8 หัวข้อ
- อนุมัติ ส่งกลับ และไม่อนุมัติ
- บันทึกโรงพยาบาลและวันติดตาม
- อัปโหลดรูปและ PDF
- แจ้งเตือน LINE, Telegram และ In-app
- สร้างรายงาน PDF

## Performance Tests
- ผู้ใช้พร้อมกันตามเป้าหมาย
- หน้าแรกเปิดเร็วบนมือถือ
- ไฟล์ขนาดใหญ่ไม่ทำให้หน้าค้าง
- Query มี Index และไม่โหลดข้อมูลเกินจำเป็น

## Release Gate
ห้ามเปลี่ยนเป็น Production จนกว่าการทดสอบระดับ Critical และ High ผ่านทั้งหมด
