# Skill: Notification Hub

## หน้าที่
รับเหตุการณ์จากระบบและส่งแจ้งเตือนไปยัง In-app, LINE และ Telegram ตามประเภทผู้รับ

## ตัวอย่างเหตุการณ์
- นักเรียนส่งความดีใหม่
- ความดีได้รับอนุมัติหรือส่งกลับแก้ไข
- เอกสารพร้อมเปิดดู
- มีวันติดตามอาการ
- ข่าวสำคัญ
- งานค้างประจำวัน

## หลักการ
- ข่าวทั่วไปแสดงในแอปก่อน
- LINE ใช้กับข้อความรายบุคคลหรือเหตุสำคัญ
- Telegram ใช้กับงานของเจ้าหน้าที่
- ข้อความสุขภาพต้องไม่แสดงรายละเอียดอ่อนไหว
- ทุกการส่งมีสถานะ pending, sent, failed หรือ skipped
- จำกัดจำนวนการส่งซ้ำและป้องกันข้อความซ้ำ

## ข้อมูลที่บันทึก
- event id
- recipient id
- channel
- status
- sent time
- retry count
- error summary
