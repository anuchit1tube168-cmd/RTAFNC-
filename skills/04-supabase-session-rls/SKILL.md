# Skill: Supabase Session and RLS

## หน้าที่
สร้างเซสชันที่เชื่อถือได้และบังคับสิทธิ์ระดับแถวให้ผู้ใช้เห็นเฉพาะข้อมูลที่ได้รับอนุญาต

## หลักการ
- ใช้ auth user ID เชื่อมกับ line account และ student UUID
- ทุกตารางข้อมูลส่วนบุคคลเปิด Row Level Security
- นักเรียนอ่านและแก้เฉพาะข้อมูลของตนเอง
- เจ้าหน้าที่เห็นข้อมูลตาม Role และ Scope เท่านั้น
- ข้อมูลโรงพยาบาลใช้ Policy เข้มกว่าข้อมูลความดี

## ขั้นตอนตรวจสอบ
1. ยืนยันว่า auth.uid() มี line account ที่ active
2. หา current student UUID
3. ตรวจ Role และ Scope
4. ใช้ Policy ในฐานข้อมูล ไม่พึ่งการซ่อนปุ่มหน้าเว็บ
5. ปฏิเสธคำขอที่ไม่มีสิทธิ์
6. บันทึก Audit Log สำหรับการเข้าถึงข้อมูลอ่อนไหว

## Test Cases
- นักเรียน A อ่านข้อมูลนักเรียน B ไม่ได้
- ผู้ตรวจความดีอ่านข้อมูลโรงพยาบาลไม่ได้
- เจ้าหน้าที่แพทย์เปิดข้อมูลสุขภาพตาม Scope ได้
- Publishable key อย่างเดียวอ่านข้อมูลส่วนตัวไม่ได้
