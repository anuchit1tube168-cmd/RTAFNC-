# Skill: Student Registry

## หน้าที่
จัดการทะเบียนนักเรียน การนำเข้าข้อมูล การตรวจรายการซ้ำ และสถานะนักเรียน

## ข้อมูลหลัก
- student UUID
- รหัสนักเรียน
- ชื่อและคำนำหน้า
- รุ่น
- ชั้นปี
- ปีการศึกษา
- สถานะการศึกษา

## ขั้นตอน
1. รับไฟล์จากผู้ดูแลระบบ
2. ตรวจหัวตารางและรูปแบบข้อมูล
3. ตรวจรหัสนักเรียนซ้ำ
4. แสดงรายการผิดพลาดก่อนยืนยัน
5. นำเข้าหรืออัปเดตข้อมูลแบบ transaction
6. สรุปจำนวนสำเร็จ ข้าม และผิดพลาด
7. สร้างรหัสเปิดใช้งานรายบุคคลเมื่อได้รับอนุมัติ

## กฎ
- ไม่เก็บไฟล์ทะเบียนจริงใน GitHub
- ใช้ student UUID เป็นความสัมพันธ์ภายใน
- บัญชีที่มีประวัติให้เปลี่ยนสถานะแทนการลบ
- การแก้รหัสนักเรียนต้องมี Audit Log
