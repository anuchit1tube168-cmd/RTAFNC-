# Skill: Incident Response

## หน้าที่
รับมือเหตุผิดปกติ เช่น บัญชีถูกใช้ผิดคน ข้อมูลรั่ว ไฟล์เปิดไม่ได้ ระบบส่งข้อความซ้ำ หรือฐานข้อมูลมีปัญหา

## Severity
- Critical: ข้อมูลสุขภาพหรือข้อมูลส่วนตัวอาจรั่วไหล
- High: มีการเข้าถึงผิดสิทธิ์หรือระบบยืนยันตัวตนผิดปกติ
- Medium: ฟังก์ชันหลักใช้งานไม่ได้บางส่วน
- Low: ปัญหา UI หรือข้อความไม่ถูกต้อง

## ขั้นตอน
1. ระบุเหตุการณ์และเวลาที่เริ่ม
2. หยุดผลกระทบ เช่น ปิดโมดูล ถอน Session หรือหยุด Queue
3. เก็บ Log และหลักฐานที่จำเป็น
4. เปลี่ยนค่าการเชื่อมต่อที่อาจได้รับผลกระทบ
5. ตรวจขอบเขตข้อมูลและผู้ใช้
6. แก้ไขและทดสอบใน Staging
7. Deploy พร้อม Rollback Plan
8. สรุปสาเหตุและเพิ่มมาตรการป้องกัน

## Rules
- ห้ามลบ Audit Log เพื่อปกปิดเหตุการณ์
- ห้ามส่งรายละเอียดสุขภาพในช่องทางแจ้งเหตุทั่วไป
- ทุกเหตุ Critical ต้องมีรายงานหลังเหตุการณ์
