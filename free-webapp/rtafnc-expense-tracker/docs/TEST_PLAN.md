# Test Plan: RTAFNC Expense Tracker

## เป้าหมายการทดสอบ

ตรวจว่าระบบทำงานครบ 4 แกนหลัก:

1. Login
2. Role / Permission
3. CRUD รายการรายรับรายจ่าย
4. Report / Export

## ชุดทดสอบขั้นต่ำก่อนใช้งานจริง

### 1. Login

- เปิดเว็บจากมือถือ
- Login ด้วย Gmail สำเร็จ
- ตรวจว่า collection `users` ถูกสร้างอัตโนมัติ
- ผู้ใช้ใหม่ต้องเป็น role `staff`

### 2. Admin คนแรก

- แก้ role ตัวเองใน Firestore เป็น `admin`
- Refresh เว็บ
- ต้องเห็นเมนู `ตั้งค่า`
- ต้องเห็นปุ่มอนุมัติ/ไม่อนุมัติในรายการรออนุมัติ

### 3. Staff

- Login ด้วยบัญชี staff
- เพิ่มรายการรายจ่าย 1 รายการ
- สถานะต้องเป็น `pending`
- staff ต้องเห็นเฉพาะรายการของตัวเอง
- staff ต้องไม่สามารถอนุมัติรายการเองได้

### 4. Admin Approve

- Login ด้วย admin
- ต้องเห็นรายการของ staff
- กดอนุมัติแล้ว status เปลี่ยนเป็น `approved`
- กดไม่อนุมัติแล้ว status เปลี่ยนเป็น `rejected`
- Dashboard ต้องนับเฉพาะรายการ `approved` ในรายรับ/รายจ่าย

### 5. Settings

- admin แก้ชื่อองค์กร
- admin แก้ปีงบประมาณ
- admin เพิ่ม/ลบหมวดหมู่
- form ต้องใช้หมวดหมู่ใหม่ทันที
- staff ต้องอ่าน settings ได้ แต่แก้ไม่ได้

### 6. Report

- เปิดหน้า Report
- ตรวจยอดรวมรายรับ รายจ่าย คงเหลือ
- ตรวจกราฟ 6 เดือนล่าสุด
- กด Print / Save PDF
- กด Export CSV แล้วเปิดไฟล์ได้ ภาษาไทยไม่เพี้ยน

### 7. Security Rules

- ห้ามใช้ข้อมูลจริงจนกว่าจะทดสอบผ่าน
- staff ต้องเขียนรายการเป็น pending เท่านั้น
- viewer ต้องดูได้อย่างเดียว
- auditLogs ต้องสร้างได้ แต่แก้ไข/ลบไม่ได้

## เกณฑ์ผ่าน

ระบบผ่าน MVP เมื่อ:

- ใช้ได้บนมือถือจริง
- Login ได้อย่างน้อย 3 role
- staff/admin/viewer แยกสิทธิ์ถูกต้อง
- export CSV ได้
- save PDF ผ่าน browser ได้
- ไม่มี `.env` จริงใน GitHub
