# LLM Wiki — dashboard-design
- **dashboard = เครื่องมือตัดสินใจ ไม่ใช่รายงานสวย**: ทุก pixel ต้องช่วยตอบคำถาม
- **F-pattern**: ตาคนกวาด ซ้ายบน→ขวา→ลงล่าง — ของสำคัญไว้ต้นทาง
- **เกณฑ์สี**: ตัวเลขเดี่ยวไม่มีความหมายถ้าไม่มีเส้นแบ่งดี/แย่
- **ข้อมูลจริงตอน mock**: scale จริงเผยปัญหา (เช่นชื่อยาวล้น, ค่า outlier ทำกราฟแบน)
