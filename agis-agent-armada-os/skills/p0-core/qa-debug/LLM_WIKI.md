# LLM Wiki — qa-debug
- **แก้อาการ vs แก้เหตุ**: try/catch กลบ error = แก้อาการ, หาว่าทำไม error = แก้เหตุ
- **reproduce ก่อนเสมอ**: bug ที่เกิดซ้ำไม่ได้จะกลับมาหลอกตอน production
- **fix one part**: หลัก Fable "ซ่อมทีละส่วน" — ทำให้ receipt ชัดว่าอะไรได้ผล
- **known-bugs.md**: ความจำสถาบัน — bug เดิมห้ามเสียเวลาแก้รอบสอง
