# SENIOR_SKILL_RULE — Skill ที่มีประโยชน์มากและใครทำแทนยาก

## Purpose
กำหนดเกณฑ์ว่า Skill MD ไหนเป็นระดับ Senior/Master/Legendary และควรทำให้ Agent Level Up ทันทีหรือไม่

## Skill Rank

### Common Skill
ใช้ได้ทั่วไป ทำตาม checklist ได้ ไม่ซับซ้อน

### Useful Skill
ช่วยลดเวลาทำงานชัดเจน ใช้ซ้ำได้หลายงาน

### Senior Skill
แก้ปัญหายากระดับ A/S ได้ มี pattern ชัด และสอน agent อื่นได้

### Master Skill
ทำงานแทนคนเชี่ยวชาญได้บางส่วน มี validation/receipt/test ครบ

### Legendary Skill
เป็น skill ที่สร้างระบบใหม่หรือทำให้ทีมทำงานแบบที่เดิมทำไม่ได้ ใครทำแทนยาก ต้องมี clone protocol

## Senior Criteria
Skill จะเป็น Senior ได้เมื่อผ่าน 5 ข้อขึ้นไป:
- มี SKILL.md ชัดเจน
- มี LLM_WIKI.md สั้นให้ LLM อ่านเร็ว
- มี CHECKLIST หรือ TEST_PLAN
- มีตัวอย่างใช้งานจริง
- มี receipt จากงานจริง
- มี risk/guardrail
- มี prompt หรือ template ใช้ซ้ำ
- ใช้ได้ข้ามบริบท เช่น report + webapp + SEO

## Legendary Criteria
ต้องผ่านทุกข้อ:
- ใช้แก้ Job ระดับ S/SS ได้
- สร้าง output ที่คนอื่นตรวจได้
- ทำซ้ำได้โดย clone agent
- มี evidence มากกว่า 1 งาน
- มี version history ใน GitHub
- มี owner agent และ backup agent
- มี approval/risk rule

## XP
- Useful Skill learned: +100
- Senior Skill learned: +300
- Master Skill learned: +600
- Legendary Skill learned: +1000
- Skill used to complete real Job: +50% bonus

## Rule
การอ่าน Skill เฉย ๆ ไม่ได้ Level Up มาก ต้องใช้ Skill ทำงานจริงสำเร็จและมี Receipt
