# 02_DECISION_ENGINE — OODA / PDCA / SWOT / MiroFish

## Purpose
กำหนดวิธีตัดสินใจของ Agent Team ให้เร็ว ปลอดภัย ตรวจได้ และปรับปรุงต่อเนื่อง

## Loop Selection

### SWOT
ใช้ก่อนเริ่มงานใหญ่หรือเสี่ยงสูง
- Strengths: จุดแข็ง / resource / skill
- Weaknesses: ช่องว่าง / missing info
- Opportunities: โอกาสสร้างระบบ ทำเงิน ทำงานเร็วขึ้น
- Threats: risk, privacy, cost, wrong data, policy

### OODA
ใช้เมื่อต้องตัดสินใจเร็ว
- Observe: รับข้อมูล ปัญหา หลักฐาน
- Orient: วิเคราะห์บริบท เลือก skill/agent/risk
- Decide: เลือก action, owner, difficulty, evidence
- Act: ทำงาน ทดสอบ เก็บ receipt

### PDCA
ใช้เมื่องานต้องปรับปรุงต่อเนื่อง
- Plan: เป้าหมาย criteria owner due date
- Do: ทำ micro-phase
- Check: ตรวจ evidence / QA / rubric
- Act: แก้ ปรับ standardize update skill

### MiroFish
ใช้แตกปัญหาเป็นก้างปลา
Problem → Cause → Skill → Agent → Mission → Evidence → Result

## Autonomy Gate

### Agent ทำเองได้
- แบ่งงาน
- route งาน
- draft รายงานหรือข้อความ
- UI copy
- QA ภายใน
- สร้าง checklist
- สร้าง issue/backlog
- prototype ที่ไม่ public

### ต้องขอ Boss Approval
- ใช้เงิน
- ลบข้อมูล
- deploy public
- ส่งอีเมลหรือข้อความจริงออกภายนอก
- เปลี่ยนข้อมูลสำคัญ
- เชื่อมบัญชีจริง
- ตัดสินใจด้านกฎหมาย แพทย์ การเงิน

## Decision Output
ทุก decision ต้องมี:
- Context
- Options
- Chosen action
- Reason
- Risk
- Approval status
- Evidence needed
- Next action
