# 01_ROUTER_POLICY — Routing Policy

## Purpose
กำหนดวิธีที่สมองกลางเลือก Skill, Agent, Hat และแหล่งข้อมูลเมื่อมี Job ใหม่

## Intake Classifier
เมื่อ Boss สั่งงาน ให้จำแนกก่อนว่าเป็นงานประเภทใด:
- Build: สร้างระบบ เว็บ dashboard app prototype
- Research: หาข้อมูลเชิงลึก อ่านไฟล์ Drive/Web/GitHub
- Report: รายงาน เอกสารราชการ PDF/DOCX/HTML
- Audit: ตรวจคุณภาพ SEO คะแนน ระบบ หรือไฟล์
- Deploy: GitHub Lovable Apps Script Cloudflare
- Design: เสื้อ ภาพ แบรนด์ สื่อประชาสัมพันธ์
- Video: ตัดต่อ คลิป สคริปต์ storyboard subtitle
- Marketing: ขายของ SEO Shopee Affiliate social
- Data: big data Google Sheets evaluation analyzer
- Automation: แจ้งเตือน scheduler LINE Gmail Calendar

## Routing Steps
1. Read Boss Command
2. Create Job Card
3. Define One Piece / Gold
4. Match Job Type
5. Select Hat Set
6. Select Skill MD
7. Select Owner Agent and Support Agents
8. Select Evidence Requirement
9. Select Approval Gate
10. Create Mission Board

## Match Rules
- SOP / ระเบียบ / PDF / DOCX → Archive Eye + Jin Guard + Copy Chef
- WebApp / GitHub / Lovable → Forge Dev + Navi Map + Bug Doctor
- SEO / Content / Landing Page → Blade SEO + Copy Chef + Hook Sniper
- Shopee / Affiliate / Sales → Hook Sniper + Archive Eye + Copy Chef
- Video Editing → Captain + Navi + Copy + Sniper + Doctor + Jin
- Big Data / Google Drive / Sheet → Archive Eye + Doctor + Forge Dev
- QA / Bug / Validation → Bug Doctor + Blade SEO + Jin Guard

## Missing Data Rule
ถ้าข้อมูลไม่พอ ให้สร้าง Missing Data List และทำเฉพาะส่วนที่ตรวจสอบได้ ห้ามเดาผลลัพธ์สำคัญ

## Router Output
- Job Type
- Hat Set
- Owner Agent
- Support Agents
- Skill MD list
- Mission split
- Evidence checklist
- Approval rule
