# PRODUCTION_IP_POLICY — One Piece Mental Model vs Production UI

## Purpose
กำหนดเส้นแบ่งให้ชัดเจนระหว่างการใช้ One Piece เป็น mental model ภายใน กับการสร้างระบบใช้งานจริงที่ปลอดภัยด้านลิขสิทธิ์

## Boss Vision
Boss ต้องการกองเรือแบบอนิเมะผจญภัย มีเรือ ลูกเรือ 3D pixel ขยับได้ พูดคุยไทย/อังกฤษ เปลี่ยนฉากตาม Job/Season/Island และมีตัวละครทำงานตามบทบาท

## Important Rule
สำหรับงานใช้งานจริง / public / deploy / demo / commercial:
- ห้ามใช้ชื่อตัวละคร One Piece จริง
- ห้ามใช้หน้าตา เครื่องแต่งกาย เรือ โลโก้ ฉาก หรือ asset ที่เหมือนต้นฉบับแบบเป๊ะ
- ห้ามทำ UI ที่ผู้ใช้เข้าใจว่าเป็นสินค้า/เกม/งานทางการของ One Piece

## Allowed Production Direction
ใช้เป็นแรงบันดาลใจเชิงระบบได้:
- Crew-based work OS
- Pirate/adventure fleet metaphor
- Island/Season mission progression
- Ship command center
- 3D pixel original characters
- Thai/English crew dialogue
- Job-based scene switching
- Level/XP/Bounty-style progress แต่ใช้ชื่อ original

## Production Promise
ทำให้ได้ความรู้สึกแบบกองเรือผจญภัย แต่เป็นจักรวาล original ของ AGIS Agent Armada OS เพื่อใช้จริงได้ในองค์กรและเผยแพร่ได้ปลอดภัยกว่า
